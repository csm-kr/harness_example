# 접근성 (Accessibility)

## 목표
- 준수 기준: {예: WCAG 2.1 AA / AAA 일부 / 자체 표준}
- 대상: {예: 키보드 사용자, 스크린 리더 사용자, 색 약자}

## 키보드 / 포커스
- 모든 인터랙션은 Tab 만으로 도달 가능해야 한다
- 포커스 표시: {예: outline 2px solid var(--focus) / 자체 토큰} — 절대 `outline: none` 금지
- 모달·드롭다운: 열릴 때 첫 인터랙션 요소로 포커스 이동, 닫힐 때 트리거로 복귀
- 트랩(focus trap)이 필요한 곳: {예: 모달, 사이드 패널}

## 스크린 리더 / ARIA
- 시맨틱 HTML 우선 (`<button>`, `<nav>`, `<main>`, `<section>`)
- ARIA 는 시맨틱으로 표현 안 될 때만 보조용
- 라이브 영역: 토스트·진행 상태 → `aria-live="polite"`
- 폼: 모든 input 에 `<label>` (또는 `aria-label`) — placeholder 만으로 의미 전달 금지

## 색 / 대비
- 본문 대비: ≥ 4.5:1, 큰 글씨: ≥ 3:1, UI 컴포넌트 경계: ≥ 3:1
- **정보를 색만으로 전달하지 마라** — 아이콘·텍스트 병기
- 색 토큰의 대비 측정값: [UI_GUIDE.md](./UI_GUIDE.md) 의 색상 표 옆에 기록

## 이미지 / 미디어
- 이미지 `alt`: 정보 전달 시 의미 있는 문구, 장식이면 `alt=""`
- 비디오: 자막 의무 (한국어). 청각 정보가 핵심이면 텍스트 transcript

## 애니메이션
- `prefers-reduced-motion: reduce` 미디어 쿼리 존중 — 큰 애니메이션 비활성화
- 자동 재생 / 자동 갱신 5초 이상 = 정지 컨트롤 의무

## 검증 체크리스트 (PR 마다)
- [ ] 키보드만으로 모든 기능 도달 가능
- [ ] 포커스 표시 모든 인터랙션 요소에서 보임
- [ ] 라이트하우스 Accessibility 점수 ≥ 95
- [ ] 새 폼 요소: label 연결됨
- [ ] 새 색상 토큰: 대비 4.5:1 통과
- [ ] 새 애니메이션: reduced-motion 분기 있음
