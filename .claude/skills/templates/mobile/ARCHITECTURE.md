# 아키텍처

> **이 문서가 답하는 질문**: 이 앱을 *어떻게* 구성하며, 화면 트리·플랫폼 분기·푸시·백엔드 연결이 어떻게 흘러가는가? *무엇을·왜* 는 [PRD.md](./PRD.md), 결정 근거는 [ADR.md](./ADR.md).

## 시스템 구조
```
[디바이스]                                    [서버 / 외부]
┌────────────────────────────────────┐
│  앱                                │      ┌──────────────┐
│  ┌──────────┐  ┌──────────┐        │ ──▶  │ 자체 백엔드  │
│  │Auth Stack│ ─▶ Main Tab │        │      │ 또는 Firebase│
│  │로그인    │  │{탭 1·2·3}│        │ ◀──  └──────────────┘
│  └──────────┘  └────┬─────┘        │
│                     ▼              │      ┌──────────────┐
│                ┌─────────┐         │ ◀──  │ 푸시 서버    │
│                │ Detail  │         │      │ APNs / FCM   │
│                │{상세}   │         │      └──────────────┘
│                └─────────┘         │
└────────────────────────────────────┘
   ▲                          ▲
   │ iOS                      │ Android  (플랫폼 분기)
```
> 화면이 늘면 Detail 박스 아래에 추가. 외부 서비스가 늘면 우측 박스 추가.

```
src/
├── screens/           # 화면 단위 컴포넌트
├── components/        # 재사용 UI
├── navigation/        # 네비게이션 정의

├── store/             # 상태 관리
├── hooks/             # 커스텀 훅
└── utils/             # 헬퍼
```

## 앱 시작 흐름
1. 앱 실행 → 토큰 저장소 조회
2. 토큰 유효 → Main Tab 진입
3. 토큰 무효/없음 → Login 화면
4. 로그인 성공 → 토큰 저장 → Main Tab
5. Main 에서 항목 선택 → Detail Stack push
6. Detail 에서 뒤로가기 → Main 복귀

플랫폼 분기:
- iOS: APNs 토큰 등록, 권한 다이얼로그 비동기
- Android: FCM 토큰 등록, 13+ 알림 권한 명시 요청
