- 빈도 정책: {예: 일 N회 상한 — 마케팅 폭주 방지}

## 통합 구조
- iOS: APNs (Apple Push Notification service)
- Android: FCM (Firebase Cloud Messaging)
- 자체 백엔드 → 양쪽으로 분기 발송 (또는 FCM 으로 통합 발송 — Android 직접, iOS 는 APNs 위임)

## 토큰 관리
- 앱 시작 시 토큰 등록 → 자체 백엔드에 저장
- 토큰 갱신 이벤트 (iOS `didRegister`, Android `onNewToken`) → 즉시 백엔드 동기화
- 로그아웃 시 토큰 무효화 (백엔드 측에서 그 사용자 ↔ 토큰 매핑 제거)
- 토큰 만료/실패: {예: 7일 무응답 → 자동 정리}

## 권한 요청
- iOS: 첫 진입에 즉시 묻지 마라 — 사용자가 가치를 인지한 시점에 (예: 첫 거래 직후)
- Android 13+: 명시적 `POST_NOTIFICATIONS` 권한 요청 화면
- 거절 후 재요청: OS 가 막음. 설정으로 안내하는 화면 필요

## 메시지 페이로드
- 공통 필드: `{ "type": "{도메인_이벤트}", "id": "{리소스 id}", ... }`
- 딥링크: `app://{화면}/{id}` 형식 — 앱이 죽어 있어도 해당 화면 직진
- 디버깅: 페이로드 로그를 `services/push.ts` 한 곳에서

## 정책 / 사용자 보호
- 마케팅 알림: 옵트인 의무 (한국 정보통신망법). 야간 (21:00~08:00) 발송 금지
- 사용자가 알림 끄면 즉시 백엔드에 반영 → 발송 큐에서 제외
